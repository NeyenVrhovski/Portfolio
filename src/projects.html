Mis Proyectos

Del Parque

Descripción general

Del Parque es una empresa de bienes raíces con origen en México. Ofrecen diversos proyectos de construcción para que los clientes residan o inviertan. Esencialmente es una página estática y serverless a excepción de el formulario de contacto, el cual está integrado con un servicio de mensajería vía mail. Particularmente yo desarrollé toda la funcionalidad del cotizador. El mismo cuenta con una integración a una API de la cual recibimos toda la información de departamentos y tipologías, para posteriormente mostrarlos en pantalla. También cuenta con la opción de generar tu propio esquema de pagos para el departamento en particular que elijas, para luego generar un PDF con los datos o enviarlos por mail. Podés ver el proyecto acá.

Tecnologías usadas

El proyecto está construido con Next JS, utilizando también Typescript. Para aplicar estilos utiliza el preprocesador SASS

Puertos

Descripción general

Puertos es una landing page administrada por Consultatio S.A., una empresa de bienes raíces. Puntualmente Puertos es uno de los barrios que Consultatio administra. La landing page de Puertos tiene una integración con la API de google maps para mostrar, en distintos mapas, los distintos lotes que tiene el barrio en cuestión. También esté integrada con un backoffice desde donde el personal de Consultatio puede administrar las imágenes, las descripciones y los precios que se muestran en la página. Particularmente yo trabajé en la parte visual de la landing, y ayudando circunstancialmente a integrar el backoffice con la página.Podés ver el proyecto acá.

Tecnologías usadas

Al ser únicamente una landing page con pocas funcionalidades, la página está hecha principalmente con HTML y CSS, utilizando la herramienta de los archivos .HBS para llevar a cabo una mejor modularización de los componentes de la página. Para las pocas integraciones y funcionalidades que tiene se utilizó JavaScript nativo. Es una aplicación monolítica, ya que tanto la landing page (el front) como el backoffice desde donde se carga la información que se muestra en la landing están en el mismo repositorio.

Club Qualia

Descripción general

Club Qualia es una plataforma de cupones para los clientes de Qualia Seguros. En ella los clientes de Qualia pueden buscar entre numerosos descuentos en distintos rubros y locales por todo Buenos Aires, tanto para canjearlos Presencialmente como online. La página está integrada mediante una API a un backoffice, también administrado por Qualia, en el cual el personal de la empresa puede cargar y borrar cupones para que luego aparezcan en el front. Particularmente yo trabaje en la parte visual de la página. Maquetando y haciendo cambios visuales a medida que el cliente lo iba solicitando. También circunstancialmente modifiqué algunas cosas de funcionalidad como la integración con la API. Podés ver el proyecto acá.

Tecnologías usadas

El proyecto está construido con Next Js, utilizando Bootstrap y SASS para la parte visual y de maquetación.

Shell Box

Descripción general

Shell Box es la pagina web de fidelidad de Shell, donde los clientes regulares de Shell pueden acumular puntos comprando sus productos y luego canjearlos por regalos. Tambien tienen una gran variedad de beneficios gratis que el usuario puede canjear en el momento para disfrutar de grandes descuentos. Tambien cuenta con un conversor en el cual puede cambiar sus puntos ShellBox por millas del programa LatamPass y viceversa. Podés ver el proyecto acá.

Tecnologías usadas

La web esta construida con Angular y utilizando SASS para los estilos. Tiene integración con gran cantidad de microservicios para, por ejemplo, recibir los beneficios, canjearlos, convertir los puntos ShellBox, etc...

Truco Posta

Descripción general

Truco Posta es un juego web y con aplicación mobile para jugar al truco en tiempo real. Tiene la novedad de poder ver y escuchar al oponente y a tus compañeros a traves del sistema de video y voz, haciendo que sea muy parecido a una partida de truco en persona. El usuario puede crear una cuenta, agregar amigos y jugar. Tambien puede pagar una suscripcion para recbir porotos mensuales, los cuales son la moneda necesaria para poder jugar. Podés ver el proyecto acá.

Tecnologías usadas

La parte front-end del juego esta conformada por tres webs distintas. La web principal (donde el usuario agenda partidas, agrega amigos, etc...) esta construida en Angular, y tiene integraciones por ejemplo con Mercadopago para las suscripciones. El lobby (en el que el usuario efectivamente juega al juego) esta hecho en React Js, y tiene integraciones tanto con Twilio (proovedor de voz y video web) como con Godot (empresa argentina de desarrollo de videojuegos) para ejecutar el juego en la plataforma web. Y finalmente la landing, la cual es la pagina de presentacion del juego, que esta orientada mas hacia el marketing. La misma esta hecha con Next Js
